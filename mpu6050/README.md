# MPU6050

## 这个工程能学什么
这是一个 MPU6050 六轴传感器（三轴加速度 + 三轴陀螺仪 + 温度）实验，用面向对象的方式封装了完整的 I2C 传感器驱动。
他演示了如何用一次 I2C 突发读取（Burst Read）获取全部 14 字节传感器数据，保证加速度和陀螺仪来自同一采样时刻；还支持轮询和 INT 中断两种数据获取模式，主循环写法完全统一。

## 运行后你会看到什么
- 串口打印 "data: ax,ay,az,gx,gy,gz" 格式的实时六轴数据
- 若芯片未检测到（WHO_AM_I 校验失败），打印 "MPU6050 error!" 并停机

## 代码做了什么
主流程是：
1. 创建 MPU6050 对象
2. 创建配置选项（绑定 I2C 句柄、量程 ±2000°/s / ±2g、DLPF 42Hz、采样率 100Hz、INT 模式）
3. 注册 MPU6050 对象（构造函数自动完成：软件复位 → 唤醒 → 配置量程/滤波器/采样率 → 重算转换系数 → 配置中断）
4. 验证 WHO_AM_I 芯片 ID
5. 在 EXTI 回调中调用 `int_handler()` 置数据就绪标志
6. 在主循环中用 `is_data_ready()` 检查 → 突发读取原始数据 → 转换为物理量 → 串口打印

## 建议重点看哪几段
- BSP/MPU6050 下的驱动代码（I2C 寄存器读写 + 量程/DLPF/采样率配置流程）
- `mpu6050_read_all_raw()` 中的突发读取——一次 I2C 事务读 14 字节，利用芯片地址自增特性
- `compute_scales()` 中量程到转换系数的映射（参考数据手册 LSB/g 和 LSB/(°/s) 表格）
- `mpu6050_is_data_ready()` 中轮询和 INT 双模式的统一接口设计（轮询永真，INT 读-清标志）
- `mpu6050_int_handler()` 的 ISR 安全设计——仅置 `volatile` 标志，不做 I2C 通信

## 你会学到的知识点
- I2C 通信协议（`HAL_I2C_Mem_Write` / `HAL_I2C_Mem_Read`，寄存器地址 + 数据读写）
- MPU6050 的初始化流程（复位 → 唤醒 → 量程 → DLPF → 采样率 → 时钟源选择）
- I2C 突发读取（Burst Read）——芯片地址自增，一次事务获取全部传感器数据
- 原始 ADC 值到物理量的转换（LSB/g、LSB/(°/s)、温度公式 `raw/340 + 36.53`）
- 数字低通滤波器（DLPF）的配置与选型（截止频率 vs 延迟的权衡）
- INT 中断模式 + ISR 安全设计（中断只置标志，主循环做 I2C 通信）
- 轮询与 INT 双模式的统一主循环接口（`is_data_ready()`）
- 动态量程切换（运行时改量程，自动重算转换系数）
- 大端序字节拼接（`int16_t = high << 8 | low`）
- 面向对象风格的 IMU 传感器驱动封装
