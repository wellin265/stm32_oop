# WS2812

## 这个工程能学什么
这是一个 WS2812/WS2812B RGB LED 灯带驱动实验，用面向对象的方式封装了 PWM + DMA 硬件时序生成驱动。
他演示了一种巧妙的单线通信实现——不靠 bit-bang（软件翻转 GPIO），而是用定时器的 PWM 占空比来编码逻辑 1 和逻辑 0，通过 DMA 将整个位流自动推送到定时器，全程 CPU 零干预。

## 运行后你会看到什么
- 4 颗 LED 依次展示：全部绿灯 2 秒 → 红/绿/蓝/白四色 2 秒 → 半亮度 2 秒
- 进入主循环后反复播放跑马灯（每颗 LED 依次闪红→绿→蓝→白）和整组切换效果

## 代码做了什么
主流程是：
1. 分配颜色缓冲区（`led_count × 3` 字节，GRB 顺序）和 PWM DMA 缓冲区（每 bit 一个 uint16_t 占空比值）
2. 创建 WS2812 对象
3. 创建配置选项（绑定 TIM2 句柄、通道、LED 数量、两个缓冲区指针及大小）
4. 注册 WS2812 对象（构造函数自动从 TIM ARR 计算 T1H/T0H 占空比、清空颜色缓冲）
5. 用 `fill()` / `set_led()` / `clear()` 设置颜色 → `send()` 生成位流并启动 DMA → `wait_done()` 等待完成
6. 在主循环中循环播放跑马灯和整组变色效果

## 建议重点看哪几段
- BSP/WS2812 下的驱动代码（PWM + DMA 生成单线时序的核心思路）
- `ws2812_calc_timing()` 中从 ARR 自动计算 T1H/T0H 的方式——T1H = 2/3 周期，T0H = 1/3 周期
- `ws2812_send()` 中的位流生成：RGB→GRB 字节序转换 → 亮度整数缩放 → 24bit 逐位展开为 PWM 占空比
- `ws2812_set_led()` 中的 GRB 存储顺序（WS2812 原生格式是 GRB，非 RGB）
- DMA 忙检测——发送前检查 `HAL_DMA_GetState()` 防止覆盖正在传输的数据

## 你会学到的知识点
- PWM 占空比编码串行数据——用硬件 PWM 替代 bit-bang 生成精确的单线时序
- DMA 内存到外设传输——DMA 将占空比数组逐一推送到定时器 CCR 寄存器，CPU 完全释放
- WS2812 单线归零码协议（T1H ≈ 0.8µs / T0H ≈ 0.4µs / RESET ≥ 50µs）
- 从定时器 ARR 自动计算时序参数（适配任意时钟配置，无需硬编码）
- RGB→GRB 字节序转换（WS2812 原生颜色顺序）
- 整数亮度缩放（`val * brightness / 255`，无浮点运算）
- 颜色缓冲与 PWM 缓冲分离的设计（亮度修改不破坏原始颜色数据）
- DMA 忙检测与传输完成等待
- 面向对象风格的 LED 灯带驱动封装
