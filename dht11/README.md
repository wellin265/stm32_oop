# DHT11

## 这个工程能学什么
这是一个 DHT11 数字温湿度传感器实验，用面向对象的方式封装了单线通信协议驱动。
他演示了一种巧妙的双向单线通信方式——利用 GPIO 开漏输出模式（`OUTPUT_OD`），主机和从机都能拉低总线，无需像 DS18B20 那样在输入/输出模式之间来回切换。

## 运行后你会看到什么
- 通过串口每秒打印一次温湿度值，格式为 "Temp: 26C, Hum: 55%"
- 若传感器通信失败（校验不通过），该次读数被丢弃不打印

## 代码做了什么
主流程是：
1. 初始化微秒延时引擎
2. 创建 DHT11 对象
3. 创建配置选项（绑定 DATA 引脚所在的 GPIO 端口和引脚）
4. 注册 DHT11 对象（内部初始化 GPIO 为开漏输出 + 上拉，释放总线）
5. 在主循环中每秒调用一次 `read()` 读取温湿度，成功则串口打印

## 建议重点看哪几段
- BSP/DHT11 下的驱动代码（开漏输出的巧妙运用——始终输出模式却能读电平）
- `DHT11_Read()` 中完整的通信流程：起始信号 → 等待响应 → 读取 40bit → 校验
- `DHT11_ReadByte()` 中利用 30µs 延时区分 bit 0（26µs 高电平）和 bit 1（70µs 高电平）
- 超时保护的实现——每个等待循环都有 `DHT11_TIMEOUT` 防止死锁

## 你会学到的知识点
- GPIO 开漏输出（`GPIO_MODE_OUTPUT_OD`）在双向单线通信中的巧妙应用
- 为什么开漏输出可以读电平——写 `PIN_SET` 释放总线后，`HAL_GPIO_ReadPin()` 读取的是 IDR 寄存器的实际电平
- 外部上拉电阻的必要性（4.7kΩ ~ 10kΩ，开漏输出依赖外部上拉拉高总线）
- DHT11 单线通信协议：起始信号（>18ms 低电平）→ 响应（80µs 低 + 80µs 高）→ 40bit 数据
- 数据校验（checksum）：湿度高 8 + 湿度低 8 + 温度高 8 + 温度低 8 = 校验和
- 超时保护机制——避免传感器异常时程序死锁
- 面向对象风格的传感器驱动封装
