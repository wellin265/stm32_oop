# DS18B20

## 这个工程能学什么
这是一个 DS18B20 数字温度传感器实验，用面向对象的方式封装了 OneWire（单总线）协议驱动。
他演示了如何通过一根 GPIO 线实现双向通信——主机拉低总线发命令、切换为输入模式读取传感器应答，以及精确到微秒的时序控制。

## 运行后你会看到什么
- 通过串口持续打印当前温度值，格式为 "temperature = 25.75 C"

## 代码做了什么
主流程是：
1. 初始化微秒延时引擎（`delay_init(72)`，72MHz 系统时钟）
2. 创建 DS18B20 对象
3. 创建配置选项（绑定 OneWire 总线所在的 GPIO 端口和引脚）
4. 注册 DS18B20 对象（内部自动初始化 GPIO 为推挽输出、释放总线）
5. 在主循环中不断启动温度转换 → 等待 750ms → 读取温度 → 串口打印

## 建议重点看哪几段
- BSP/DS18B20 下的驱动代码（OneWire 协议的完整实现：复位脉冲、写 bit/byte、读 bit/byte）
- GPIO 模式切换（输出 PP 模式用于发命令，输入模式用于读数据）
- `ReadBit()` 中的精确定时——拉低 1µs → 切换输入 → 等待 5µs → 采样
- `ReadTemp()` 中读取 Scratchpad 寄存器的流程

## 你会学到的知识点
- OneWire（单总线）协议的时序实现——复位/应答/读写位
- GPIO 输入/输出模式的运行时切换（`HAL_GPIO_Init()` 切换 Mode）
- 微秒级精确延时的应用（`delay_us()` 实现 1µs ~ 480µs 的精确控制）
- DS18B20 的命令流程：Reset → SKIP_ROM → CONVERT_T → 等待 → Reset → SKIP_ROM → READ_SCRATCHPAD
- 温度数据的补码解析（16 位有符号整数 × 0.0625 = 实际温度）
- 面向对象风格的传感器驱动封装
