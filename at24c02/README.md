# AT24C02

## 这个工程能学什么
这是一个 AT24C02 EEPROM 存储芯片实验，用面向对象的方式封装了 I2C 通信驱动。
他演示了如何通过硬件 I2C 读写 EEPROM，以及写入时自动处理页边界——当数据跨越 8 字节页边界时自动拆分为多次 I2C 事务，每次写入后等待 5ms 内部写周期。

## 运行后你会看到什么
- 向 EEPROM 地址 0 写入 16 字节数据（值为 0xAA）
- 从 EEPROM 地址 0 读回 16 字节
- 通过串口打印读出的 16 个字节（预期全为 "aa"）

## 代码做了什么
主流程是：
1. 创建 AT24C02 对象
2. 创建配置选项（绑定 I2C 句柄、设备地址 0xA0、页大小 8 字节、容量 256 字节）
3. 注册 AT24C02 对象
4. 调用 `write()` 从地址 0 写入 16 字节测试数据
5. 调用 `read()` 从地址 0 读回 16 字节
6. 串口打印读回的数据以验证写入正确

## 建议重点看哪几段
- BSP/AT24C02 下的驱动代码（HAL I2C Memory 函数的使用方式）
- `at24c02_write()` 中的自动分页逻辑——计算当前页剩余空间、拆分数据、每页写后等 5ms
- `at24c02_check_addr()` 中的地址越界校验

## 你会学到的知识点
- I2C 通信协议（AT24C02 的设备地址与寻址方式）
- HAL I2C Memory 函数的使用（`HAL_I2C_Mem_Read` / `HAL_I2C_Mem_Write`）
- EEPROM 的页写入机制——为什么不能跨页连续写
- 自动分页写入的实现（页偏移计算 → 拆分 → 逐页写入 → 等待写周期）
- EEPROM 写周期等待（每次页写入后需 5ms 完成内部编程）
- 地址越界校验
- 面向对象风格的存储设备驱动封装
