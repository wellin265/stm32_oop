# USART_RING

## 这个工程能学什么
这是一个 UART DMA + 环形缓冲区 + IDLE 中断实验，用面向对象的方式封装了高可靠性的串口通信驱动。
他演示了如何用 DMA 收发数据、用环形缓冲区缓存接收数据、用 IDLE 中断自动识别帧边界，以及格式化发送和二进制发送两种模式。

## 运行后你会看到什么
- 通过串口助手向 MCU 发送任意数据
- MCU 收到数据后立即回显 "received data: xxx"（DMA + 环形缓冲区自动接收，不丢数据）

## 代码做了什么
主流程是：
1. 创建 USART_RING 对象
2. 创建配置选项（绑定 UART 句柄、用户提供的收发缓冲区及大小、波特率）
3. 注册 USART_RING 对象（内部自动启动 DMA 接收 + IDLE 中断）
4. 在主循环中调用 `available()` 查询可读字节数
5. 有数据时调用 `read()` 读取并回显

## 建议重点看哪几段
- BSP/USART_RING 下的驱动代码（环形缓冲区的 head/tail 指针管理、DMA 接收 + IDLE 中断处理）
- `send()` / `send_raw()` 中的 DMA 忙检测机制——避免覆盖正在发送的数据
- `irq_handler()` 中 IDLE 中断如何识别帧边界、更新环形缓冲区写指针
- 环形缓冲区读写中的关中断保护

## 你会学到的知识点
- UART + DMA 的配合使用（CPU 零干预接收）
- 环形缓冲区（Ring Buffer）的经典实现（head/tail 指针）
- IDLE 中断检测帧边界——硬件自动识别一帧数据接收完毕
- DMA 发送的忙检测机制——避免静默丢帧
- 格式化发送 `send()` 与二进制发送 `send_raw()` 的实现
- 关中断保护共享数据（ISR 与主循环并发访问）
- 面向对象风格的通信驱动封装
- 为 BLE OTA 固件更新等高可靠性场景打基础
