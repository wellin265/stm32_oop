# SORT_TIMER

## 这个工程能学什么
这是一个软件定时器实验，在 TIM2 硬件定时器 1ms 时基之上，用面向对象的方式封装了一个软件定时器管理器。
他演示了如何注册多个软件定时器，支持单次触发和周期性自动重载两种工作模式，以及 ISR 与主循环的协作方式。

## 运行后你会看到什么
- 通过串口每隔 500ms 打印一次 "500ms elapsed!!!"
- 3 秒后打印一次 "3 seconds elapsed!"（仅一次）
- 后续持续每 500ms 打印周期性消息

## 代码做了什么
主流程是：
1. 创建软件定时器管理器对象
2. 创建定时器配置（指定最大定时器数量）
3. 注册两个定时器：一个 500ms 自动重载模式，一个 3 秒单次模式
4. 启动 TIM2 硬件定时器产生 1ms 中断
5. 在 ISR 中调用 `tick()` 驱动所有定时器计数
6. 在主循环中轮询 `check()` 判断超时并执行对应操作

## 建议重点看哪几段
- BSP/SORT_TIMER 下的驱动代码（定时器池管理、两种模式的超时处理）
- main.c 中 ISR 回调与主循环的配合
- `start()`/`stop()`/`check()` 中的临界区保护（`__disable_irq()`）

## 你会学到的知识点
- 软件定时器的设计与实现（基于硬件时基的纯软件扩展）
- 单次模式（ONCE）与自动模式（AUTO）的区别和实现
- ISR 上下文与主循环上下文的职责分离
- 临界区保护（`__disable_irq()` / `__enable_irq()`）确保数据一致性
- 定时器池的静态管理——无动态内存分配
- 函数指针实现面向对象风格的 API 封装
